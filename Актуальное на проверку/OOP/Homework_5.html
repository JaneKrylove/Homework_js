1. Изучите документацию по работе с модулями в js тут. Проработайте примеры, приведенные в статье, самостоятельно. Выполнено. 2. Какие практические задачи позволяют решать модули? Модули позволяют осуществить импорт/экспорт частей кода в один или несколько файлов, позволяют инициализировать/настраивать объекты. 3. Все ли браузеры на текущий момент поддерживают работу с модулями? Большинство браузеров поддерживают модули. Наверное за исключением IE. 4. Зачем нужно расширение .mjs? Это файл исходного кода JavaScript, который используется в качестве модуля ECMA (модуль ECMAScript) в приложениях Node. js. 5. Что такое импорт и экспорт? export отмечает переменные и функции, которые должны быть доступны вне текущего модуля. import позволяет импортировать функциональность из других модулей. 6. В скриптах какого типа можно работать с модулями? В современном JavaScript осталось два основных стандарта модульных систем. Это CommonJS, которая является основной для платформы Node.js, и ESM (ECMAScript 6 модули), которая была принята как стандарт для языка и внесена в спецификацию ES2015. 7. Что такое “use strict”? Директива «use strict» (перевод: «использовать строгий») позволяет писать код в более строгом режиме. Это означает, что интерпретатор будет проявлять больше внимания к потенциальным ошибкам в коде.
